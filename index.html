Local BE App Setup - All commands prefixed with working directory
📁 Working Directory: C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\
🧹 Step 1 — Stop IIS 
IISRESET /stop
🗑️ Step 2 — Clean Solutions (optional) 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Clean-KarbonApplication.ps1
💡 Tip: Cleans all solution bin folders. Not always required, but helps resolve stubborn build issues.
🗄️ Step 3 — Publish Database Externals (optional) 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonBuildAndPublishDatabaseExternals.ps1
💡 Tip: Publishes shared DB dependencies (externals). Good to run if you haven't done so in a while, or if DB publish is failing.
🗄️ Step 4 — Publish All Databases (required) 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonBuildAndPublishDatabases.ps1
💡 Tip: Publishes all Karbon pillar databases. This is usually the step with the most issues.
💡 Manual Fix: If a specific pillar DB fails, open its .sln in Visual Studio as admin, right-click the DB project → Publish, and set:

• Server Name: .
• Database: <PillarName> (e.g. Tenant, Work)
• Trust Server Certificate: True

Then re-run the script.
💡 Fast Alternative: If you only need a quick DB sync after merging develop: 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonBuildAndPublishDatabasesFast.ps1
🏗️ Step 5 — Build & Publish All Pillars (required) 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonPillarsNoDatabases.ps1
💡 Faster Alternatives (compile-check only, no publish):

Compile only (with package restore): 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonPillarsNoDatabases.NoPublish.ps1
Fastest compile (no restore, no publish): 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Build-KarbonPillarsNoDatabases.NoRestore.NoPublish.ps1
⚙️ Step 6 — Configure IIS (first-time setup only) 
C:\Dev\BalancedLabs\DeploymentUtilities\Snipets\Configure-KarbonIIS.ps1
💡 Tip: Only needed after a .NET 8 migration or first-time setup. Configures and populates all required IIS sites.
